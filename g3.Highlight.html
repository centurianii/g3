Highlight

g3. Highlight

new Highlight()

A wrapper class for different highlight projects.

Source:
  • plugins/g3Highlight/g3Highlight.js, line 2
Version:
  • 0.1
Author:
  • https:/github.com/centurianii
Copyright:
  • MIT licence

It loads the project's files and handles the nodes according to the requirements of the library.

A note on libraries: before switching to a new one call a this.destroy() to get rid off previous files.

A note on languages and themes: user arguments should contain the first word (up to a . character) of the last part of a filepath/url, see g3.Highlight.resources and g3.Highlight.resourceFilterKeys.

A note on extending this class: all switch...case blocks in code that use the keys from g3.Highlight.resources.

It supports:

  • Highlight,

  • rainbow,

  • add yours!.

(static) resourceFilterKeys :Object

A collection of key-values per library used in resource filtering.

Source:
  • plugins/g3Highlight/g3Highlight.js, line 112

The keys of this object are the same as those in g3.Highlight.resources and every key represents a library with value an object used to filter the resources:

{
   languages: <2nd string from last in paths> ,
   themes: <2nd string from last in paths>
}

If these keys for a library, namely languages and themes, fail to correlate with the relevant strings in g3.Highlight.resources the code fails to load the proper resources!

Value of keys:

  • libraries represents the last folder in a language filepath

  • themes represents the last folder in a theme filepath.
